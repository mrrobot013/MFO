# Парсер МФО

Программа открывает сайт Web-Zaim по партнёрской ссылке, заполняет заявку, записывает SMS и звонки в файл **`data/sms_log.xlsx`**.

Нужны: Mac, Python 3.11, Google Chrome и телефон для приёма SMS.

Поддерживаются два автоматических режима:

- **iPhone Forward SMS (SMS авто)** — iPhone передаёт входящие SMS в парсер через приложение Forward SMS. Звонки с iPhone автоматически не передаются из-за ограничений iOS.
- **Android SMS + звонки (webhook)** — Android через MacroDroid передаёт в парсер и SMS, и входящие звонки: номер, время, статус/тип и длительность.

---

## 1. Установка (один раз)

Открой **Терминал** и выполни команды по порядку (копируй блоками).

Перейди в папку проекта (путь подставь свой, если папка лежит в другом месте):

```bash
cd ~/Desktop/Гидфинанс/mfo_parser
```

Создай окружение и установи зависимости:

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Создай файл настроек:

```bash
cp .env.example .env
```

---

## 2. Запуск интерфейса (каждый раз)

В Терминале:

```bash
cd ~/Desktop/Гидфинанс/mfo_parser
source .venv/bin/activate
./gui.sh
```

Откроется **Safari** с панелью управления (`http://localhost:8501`).

Терминал не закрывай — пока он открыт, работает интерфейс. Остановить: `Ctrl+C` в терминале.

> Интерфейс — в Safari. Chrome парсер использует отдельно.

## 3. Настрой телефон

### Вариант A: iPhone для SMS

На iPhone поставь и настрой **Forward SMS** так, чтобы входящие SMS уходили на HTTP URL парсера.

iPhone и Mac должны быть в одной Wi-Fi сети.

В интерфейсе парсера в шаге 2 будет показан адрес твоего Mac, например:

```text
SMS URL:  http://192.168.1.25:8765/sms
```

В Forward SMS настрой:

- все входящие SMS отправлять `POST` на `/sms`;
- формат можно JSON или form-data.

Поддерживаемые поля для SMS:

```json
{"sender":"WEBZAIM","text":"Код 1234 https://example.ru","timestamp":1710000000000}
```

Если приложение присылает отправителя как `SIM 1`, а настоящий отправитель
добавляет в текст хвостом, например `@web-zaim.ru #1234 (23.05.2026 21:38)`,
парсер сам достанет альфа-имя и дату из этого хвоста.

### Вариант B: Android для SMS + звонков

На Android поставь **MacroDroid - Device Automation**. В интерфейсе выбери режим **Android SMS + звонки (webhook)**.

В MacroDroid настрой два макроса:

- `SMS Received` → `HTTP Request POST` на `/sms`;
- `Call Incoming` или событие Call Log → `HTTP Request POST` на `/call`.

Пример SMS payload:

```json
{"sender":"WEBZAIM","text":"Код 1234 https://example.ru","timestamp":1710000000000}
```

Пример payload для звонка:

```json
{"number":"+79837210053","durationMillis":25000,"type":"3","timestamp":1710000000000}
```

Парсер понимает разные названия полей: `number`, `phoneNumber`, `caller`, `timestamp`, `dateMillis`, `duration`, `durationMillis`, `type`, `callType`.

Подробная инструкция для Android — в [ANDROID_MACRODROID.md](ANDROID_MACRODROID.md).

## 4. Полный прогон (кнопки в Safari)

**Перед началом:** выключи VPN (иначе сайт часто показывает ошибку 403).

| Шаг | Что сделать |
|-----|-------------|
| 1 | Нажми **▶ Запустить Chrome + открыть страницу МФО** (подождись, пока вверху появится «Chrome ✓ работает»). |
| 2 | Выбери режим телефона: **iPhone Forward SMS (SMS авто)** или **Android SMS + звонки (webhook)**, затем введи свой номер (`+7…`). |
| 3 | Нажми **▶ Запустить парсер (заполнить заявку)**. Парсер сам заполнит анкету и отправит. |
| 4 | SMS с кодом придёт на iPhone, Forward SMS сам отправит её в парсер. Парсер достанет код и впишет его на сайте. |
| 5 | Все следующие SMS попадут в Excel автоматически. В Android-режиме туда же попадут входящие звонки. |
| 6 | Нажми **⏹ Остановить парсер**, затем **⬇ Скачать sms_log.xlsx**. При желании **🛑 Закрыть Chrome**. |

Таблица с результатом — во вкладке **📊 Таблица (ТЗ)** внизу страницы.

---

Ручной режим оставлен как запасной: можно выбрать **Ручной ввод в интерфейсе** и вставлять SMS руками.

## 5. Где лежит результат

Файл: **`mfo_parser/data/sms_log.xlsx`**

- лист **`sms`** — текст SMS, дата, отправитель, ссылка после редиректа (4 поля из ТЗ);
- лист **`Links`** — отдельная строка на каждую ссылку из SMS: исходная ссылка, финальная ссылка, домен, HTTP-статус и цепочка редиректов;
- лист **`Alphas`** — сводка по альфа-именам отправителей;
- лист **`calls`** — входящие звонки, если используется Android/MacroDroid или другой call webhook;
- лист **`Analytics`** — общая аналитика: количество SMS, звонков, статистика по дням, отправителям и доменам.

Если в SMS только код без ссылки — колонка с URL будет пустой, это нормально.

## 6. Если что-то пошло не так

| Проблема | Что сделать |
|----------|-------------|
| На сайте **403 Forbidden** | Выключи VPN, обнови страницу МФО в Chrome, снова **Запустить парсер**. |
| **Chrome не запускается** | Закрой обычный Chrome полностью (Cmd+Q), снова нажми кнопку запуска Chrome. |
| Кнопка парсера серая | Сначала запусти Chrome (шаг 1), проверь номер телефона. |
| SMS не отправляется | Парсер должен быть запущен (вверху «Парсер ▶ работает»). |
| iPhone не достучался до Mac | Проверь, что iPhone и Mac в одной Wi-Fi сети, а URL в приложении начинается с IP Mac из интерфейса. |
| Звонки не попадают в Excel | Автоматические звонки поддержаны для Android/MacroDroid или другого webhook-источника. iPhone штатно не отдаёт входящие звонки в Shortcuts/Forward SMS. |
| `./gui.sh` пишет про `.venv` | Сначала выполни блок из раздела **Установка**. |

## 7. Запуск без интерфейса (через Терминал)

Если нужен только терминал, без Safari:

```bash
cd ~/Desktop/Гидфинанс/mfo_parser
source .venv/bin/activate

python main.py chrome    # поднять Chrome
python main.py run       # полный сценарий
```

По умолчанию `.env` настроен на webhook для Forward SMS. Парсер покажет URL `/sms` и `/call` в логах.

Если нужен ручной ввод SMS без Forward SMS, поставь в `.env`:

```ini
SMS_PROVIDER=manual
CALL_PROVIDER=off
```

Тогда, когда в логе появится «жду SMS», вставь текст SMS в **этот же** терминал и нажми Enter.

Остановить приём SMS: введи `.stop` и Enter.

Посмотреть таблицу:

```bash
python main.py show
```

Закрыть Chrome парсера:

```bash
python main.py kill-chrome
```

Подробные сценарии проверки — в [TESTING.md](TESTING.md).
